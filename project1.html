60 Days of DevOps

I documented my journey of 60 Days DevOps challenge. The reason for this documentation is to help others understand the concepts that are required for DevOps.

The repository has gotten 95 stars and 22 forks in just 1 month and it is still growing.

This journey will not cover all things about "DevOps" but it will cover the areas that I feel will benefit my learning and understanding overall.

  • I have created 60 videos for 60 days. So if you don't understand the documentation, you can watch the videos also.
  • Various tools and technologies are documented here and I have divided them in subparts so that a viewer can easily understand.

Following are the technologies that are discussed in these 60 Days DevOps.

  • Git and GitHub
  • Networking
  • Linux
  • YAML
  • Docker
  • Kubernetes
  • Prometheus
  • Terraform
  • Ansible
  • CI/CD Pipeline in Jenkins
  • Continuous Monitoring by Nagios
  • Cloud Computing in AWS
  • Helm
